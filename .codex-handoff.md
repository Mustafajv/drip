# Codex Handoff

## Current Goal

Build DRIP into a proper local full-stack store:

- Customers can sign up, sign in, place mock-checkout orders, and view their own order history.
- Admin access is role-based and restricted to one seeded admin user.
- Products and orders use Prisma/Postgres as the source of truth.
- Admin can manage the product catalog.

## Implemented State

- Added database roles with `CUSTOMER` as default and `ADMIN` for the seeded admin user.
- Added database-backed products, orders, and order items.
- Added server-side auth guards for signed-in users and admins.
- Replaced frontend product localStorage persistence with Express API calls.
- Added protected `/admin` product catalog CRUD page.
- Added protected `/account` customer order history page.
- Checkout now creates persisted orders and redirects customers to `/account`.
- Navbar shows `Account` to signed-in users and `Admin` only when the session role is `ADMIN`.
- Added `npm run seed:admin` to promote the configured `ADMIN_EMAIL` user.

## Active Decisions

- Admin access is controlled by database role, not frontend-only checks.
- Exactly one admin is expected for now, promoted via `ADMIN_EMAIL` and `npm run seed:admin`.
- Products and orders are stored in Prisma/Postgres.
- Product images remain URL-based.
- Checkout remains mock payment; no Stripe/payment provider yet.
- Cart remains client-side Zustand state until checkout.
- Customers can only see their own orders.
- Admin product scope is catalog CRUD only for now.
- No inventory tracking, saved addresses, multi-admin management, or payment integration in the current scope.
